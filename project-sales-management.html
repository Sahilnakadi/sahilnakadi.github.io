Sales Management Dashboard

Executive sales overview tracking internet sales by customer, product, and budget with daily data refresh via AdventureWorks DW.

Database
AdventureWorksDW2022
Tools Used
Power BI, SQL Server
Focus Area
KPI Tracking, Budget Analysis

🎯 Business Request & User Stories

The business request for this data analyst project was an executive sales report for sales managers. Based on the request from the business, the following user stories were defined to fulfill delivery and ensure acceptance criteria were maintained throughout the project.

# As a (role) I want (request) So that I (user value) Acceptance Criteria
1 Sales Manager Dashboard overview of internet sales Follow which customers and products sell best Power BI dashboard, daily update
2 Sales Representative Detailed overview per Customers Follow up customers who buy most Dashboard with customer filters
3 Sales Representative Detailed overview per Products Follow up products that sell most Dashboard with product filters
4 Sales Manager Dashboard of sales over time vs budget Follow sales against budget Graphs and KPIs comparing vs budget

🔧 Data Cleansing & Transformation (SQL)

To create the necessary data model for analysis and fulfill the business needs defined in the user stories, the following tables were extracted using SQL. One data source (sales budgets) was provided in Excel format and connected in the data model in a later step.

DIM_Calendar

Date dimension table cleansed to include only relevant date attributes for time-based analysis.

-- Cleansed DIM_DateTable -- SELECT [DateKey], [FullDateAlternateKey] as date, [EnglishDayNameOfWeek] as day, [WeekNumberOfYear] as weekNr, [EnglishMonthName] as month, LEFT([EnglishMonthName],3) as monthshort, [MonthNumberOfYear] as monthNo, [CalendarQuarter] as Quarter, [CalendarYear] as year FROM [AdventureWorksDW2022].[dbo].[DimDate]

DIM_Customers

Customer dimension with geographic information and full name concatenation for better reporting.

SELECT c.[CustomerKey] AS CustomerKey, [FirstName] AS [First Name], [LastName], CASE when c.[Gender] = 'm' then 'male' when c.[Gender] = 'f' then 'female' end as [gender], [DateFirstPurchase], g.[City] AS [CustomerCity], CONCAT(c.FirstName, ' ', c.LastName) AS [FullName] FROM [AdventureWorksDW2022].[dbo].[DimCustomer] c JOIN [AdventureWorksDW2022].[dbo].[DimGeography] g ON c.GeographyKey = g.[GeographyKey];

DIM_Products

Product dimension with category hierarchy and status handling for discontinued products.

SELECT [ProductKey], [ProductAlternateKey] as ProductItemCode, ps.[EnglishProductSubcategoryName] as [Sub Category], pc.[EnglishProductCategoryName] as [Product Category], p.[EnglishProductName] as [Product Name], [Color] as [Product Color], [Size] as [Product Size], ISNULL(p.Status, 'outdated') as [Product Status] FROM [dbo].[DimProduct] as p LEFT JOIN dbo.DimProductSubcategory as ps ON p.ProductSubcategoryKey = ps.ProductSubcategoryKey LEFT JOIN dbo.DimProductCategory as pc ON ps.ProductCategoryKey = pc.ProductCategoryKey;

FACT_InternetSales

Fact table filtered to include only the last 2 years of sales data for performance optimization.

SELECT [ProductKey], [OrderDateKey], [CustomerKey], [SalesOrderNumber], [SalesAmount] FROM [AdventureWorksDW2022].[dbo].[FactInternetSales] WHERE LEFT(OrderDateKey, 4) >= YEAR(GETDATE()) - 2 ORDER BY OrderDateKey ASC

🗄️ Data Model

The data model follows a star schema design with the FACT_InternetSales table at the center, connected to dimension tables (DIM_Date, DIM_Customer, DIM_Product) and the FACT_Budget table for budget comparison analysis.

Key Relationships:
  • FACT_InternetSales → DIM_Date (OrderDateKey → DateKey)
  • FACT_InternetSales → DIM_Customer (CustomerKey → CustomerKey)
  • FACT_InternetSales → DIM_Product (ProductKey → ProductKey)
  • FACT_Budget → DIM_Date (Date → DateKey)
